Step 2: Paste this Configuration
Open that new docker-compose.yml file in VS Code and paste this exact configuration inside it. This setup includes the Kafka Broker, Zookeeper (which manages Kafka), and a Kafka UI dashboard (so you have a visual interface to see your messages, just like you had a UI in Thunder Client).

YAML
version: '3.8'

services:
  zookeeper:
    image: confluentinc/cp-zookeeper:7.4.0
    container_name: zookeeper
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000

  kafka:
    image: confluentinc/cp-kafka:7.4.0
    container_name: kafka
    depends_on:
      - zookeeper
    ports:
      - "9092:9092"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1

  kafka-ui:
    image: provectuslabs/kafka-ui:latest
    container_name: kafka-ui
    ports:
      - "8080:8080"
    depends_on:
      - kafka
    environment:
      KAFKA_CLUSTERS_0_NAME: local
      KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS: kafka:9092

What is confluentinc/cp-zookeeper? Confluent is the company founded by the original creators of Apache Kafka. They maintain the most stable, secure, and well-optimized Docker images (cp stands for Confluent Platform).

Why does Kafka depend on Zookeeper? Kafka is a distributed system. Zookeeper acts as the manager/coordinator. It keeps track of which Kafka brokers are alive, handles configurations, and manages data coordination.

Are there alternatives? Yes! Newer versions of Kafka use KRaft (Kafka Raft metadata mode), which removes Zookeeper entirely. However, the Zookeeper architecture remains the industry standard for production stability and learning the core mechanics of message queues.


The Configurations & Port Numbers
Tick Time: This is not related to stocks or trading keywords! In Zookeeper, a "tick" is simply the basic time unit (in milliseconds) it uses to measure heartbeats. 2000 means 2 seconds. It's how often Zookeeper checks if Kafka is still alive.

Broker ID: If you scale your system to run 5 or 10 Kafka servers, each one needs a unique number identifier. Since we are running just one instance, it's 1.

Zookeeper Connect: This tells the Kafka service exactly how to find its coordinator container over Docker's internal network (zookeeper:2181).

Advertised Listeners & Ports (9092): 9092 is the standard default port universally assigned to Apache Kafka (just like MongoDB defaults to 27017). "Advertised Listeners" is how Kafka tells your backend application, "Hey, look for me on localhost:9092 to send messages!"

"Isn't our main on 5k tho?" Your Node.js/Express backend server runs on port 5000. That doesn't change! Your backend on port 5000 will talk to Kafka on port 9092. They run side-by-side.

Topic Replication Factor: This defines how many backup copies of your data Kafka should keep across different servers. Since we only have 1 local server running right now, replication must be set to 1.

The Visual UI Layer
Kafka UI: Because Kafka has no graphical interface by default, provectuslabs/kafka-ui sets up a web portal so you can view your data pipelines visually.

Ports 8080:8080: Port 8080 is the standard port for web dashboards (completely separate from Google or standard HTML pages). You will access it via your web browser.

Cluster Name & Bootstrap Servers: This simply tells the UI app to label your view as "local" and connect to the core Kafka system at kafka:9092.
