3. What is user._id? Why the underscore?
In MongoDB, every document automatically gets a completely unique identification code when it's created. MongoDB enforces a strict naming convention: this primary key is always named _id with a leading underscore.

What it is: It is not a function; it is a BSON ObjectType Property.

Why .toString() is used: Because MongoDB stores _id as a complex, raw binary buffer object, we run .toString() on it to flatten it into a clean text string ("65b2c...") before shipping it off to Kafka.
